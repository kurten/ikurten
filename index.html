开发日志(with node.js)

2014-03-01 | Comments

node.js开发

最近的项目一直在用node.js开发，包括web端、移动端的api实现，以及后台消息系统都是用node.js。web端和api实现都是基于express这个框架基础上去做包装实现的，逐渐构建了一个类似于rails的框架。开发人员只要注重实现controller和view以及前台页面交互就可以了，不需要考虑其他模块载入，封装什么的内容。controller实现只要遵照一种约定即可。

目标需求大致罗列如下：

  • 模块自动载入(controller、middleware、config等)
  • response.render页面时公共输出内容加载输出
  • web app的安全配置(copy了lusca)
  • 前端模板引擎要求可以使用cdn加速
  • 自动化编译发布
  • 部署方便，快速

于是采用的技术选型为:

  • nodemon(开发环境使用)
  • bower(前端依赖安装)
  • grunt(自动化打包编译发布)
  • pm2(正式环境使用)
  • async(用来解决回掉函数太多的问题，比如sql查询的回掉)
  • dustjs-linkedin(可以编译为js，用于cdn加速)
  • 自己实现了模块载入工具对象以及模板渲染输出时的hook
  • 前端基础样式UI库用bootstrap和compass，方便编写

controller实现的约定


/**
 * 遍历文件夹，获取并且绑定controller中导出的方法
 * 仅支持多级目录
 * controller导出的模式为
 * module.exports = {
 *     hello:{method:'get', path:'/hello', fun:function (req, res) { res.send('hello world'); }}
 * }
 * -------- or --------
 * exports.method = 'get';
 * exports.path = '/hello';
 * exports.fun = function (req, res) {
 *     res.send('hello world');  
 * }
 */

middleware实现的约定


/**
 * 载入自定义中间件 例如 auth等
 * 导出模式(支持单个中间件中多个导出函数以及多级目录)
 * exports.path = "/hello";   //可选选项没有默认为'*'
 * exports.auth = function(req, res, next) {
 *     next();  
 * }
 */

render hooks实现的约定


/**
 * 载入自定义hooks 例如 hookForAll.js等
 * 导出模式(支持多级目录)
 * exports.path = '*';
 * exports.fun = function (options, req, res, cb) {
 *     cb && cb(options);
 * };
 */

通过源代码安装ruby-2.0.0-p247

2013-07-06 | Comments

前两天心血来潮，想升级一下本机的ruby版本，于是下载了最新版本的ruby代码，然后


./configure 
make 
sudo make install

但是碰到一个问题，openssl can’t load。找了很多方法都搞不定，怀疑是本机openssl版本低于0.9.6b，于是重新下载了openssl的代码(1.0.1e版本)编译安装。然后再重新安装ruby，然后出来一个


In file included from openssl_missing.c:22:
openssl_missing.h:71: error: conflicting types for ‘HMAC_CTX_copy’
/usr/include/openssl/hmac.h:102: error: previous declaration of ‘HMAC_CTX_copy’ was here
openssl_missing.h:95: error: conflicting types for ‘EVP_CIPHER_CTX_copy’
/usr/include/openssl/evp.h:502: error: previous declaration of ‘EVP_CIPHER_CTX_copy’ was here
openssl_missing.c:27: error: conflicting types for ‘HMAC_CTX_copy’
/usr/include/openssl/hmac.h:102: error: previous declaration of ‘HMAC_CTX_copy’ was here
openssl_missing.c:122: error: conflicting types for ‘EVP_CIPHER_CTX_copy’
/usr/include/openssl/evp.h:502: error: previous declaration of ‘EVP_CIPHER_CTX_copy’ was here

以上错误我估计是extconf.rb配置编译header的时候，哪里出错，没有判断完全方法定义函数，导致函数重复定义了。因为对于autoconf的机制以及openssl不熟悉，所以暂时无法深究这个问题，于是简单的对openssl_missing.h打了个补丁，重新编译，问题即可解决。如果make的时候报出以上错误，直接在openssl_missing.h加入


#define HAVE_HMAC_CTX_COPY 1
#define HAVE_EVP_CIPHER_CTX_COPY 1

两行，然后make && sudo make install即可。

have fun :).

论文阅读

最近一直在读这里的论文。学到了很多东西。目前已经读完了第一部分，深入理解了Gossip Protocol、Vector Clock，并对Cassandra、BigTable、Google File System、MapReduce实现做了初步的理解。

Gossip Protocol

Gossip Protocol其实是一种传播方式，在一个集群中，A随机的取一台机器B，把A自己手中所知道的所有机器状态、机器IP等一些信息，发送给B，B收到这些信息后，和自身的内容做个比较、合并。这样就完成了一次传播。开源的实现java-gossip、gossip-protocol-java。

Vector Clock

Vector Clock，用来合并集群中有冲突的数据，但是还是要应用曾去取舍最终的数据。具体的解释我觉着这篇文章解释的很详细了。
